                  泛型

                  聂 2020-02-26 基础

                  概述： 泛型不仅是c#编程的一部分，与程序集中的IL（Intermediate Language 中间语言）代码紧密集成，有了泛型就可以创建独立于被包含类型的类和方法。

                  优势： 1.性能 2.类型安全性 3.二进制代码重用 4.代码的扩展 5. 命名约定

                  # 性能

                  值类型存储在栈上，引用类型存储在堆上。从值类型转换为引用类型称为装箱，装箱的值类型转换为值类型称为拆箱，拆箱时，使用类型强制转换运算符。装箱和拆箱操作很容易使用，但性能损失比较大，遍历许多项尤其如此，泛型能避免频繁的装箱拆箱。

                  # 类型安全

                  如果使用ArrayList集合中添加一个整数，字符串，其他类型的对象，使用foreach语句遍历会出先运行异常，在泛型类List<T>中，泛型类型T定义了允许使用的类型，定义了T类型如 List<int>编辑器就只能把整型添加到集合中。

                  # 二进制代码重用

                  泛型类可以定义一次，并且可以使用许多不同的类型实例化。

                  # 代码的扩展

                  在用不同的特定类型实例化泛型时，会创建多少代码？因为泛型类的定义会放在程序集中，所以用特定类型实例化类不会在IL代码中复制这些类，但是，在JIT编辑器把泛型类编译为本地代码时，会给每个值类型创建一个新类。引用类型共享同一个本地类的所有相同的实现代码。这是因为引用类型在实例化的泛型类中只需要4个字节的内存地址（32位系统），就可以引用一个引用类型。值类型包含在实例化的泛型类的内存中的，因为每个值类型对内存的要求都不同，所以要为每个值类型实例化一个新类。

                  # 命名约定

                  1. 反省类型的名称用字母T作为前缀。
                  2. 如果没有特殊要求，泛型类型允许用任意类替代，且只使用了一个泛型类型，就可以用字符T作为泛型类型的名称。

                  # 创建泛型类

                  一般的、泛型的简化链表类。

                    //创建节点类
    public class LinkedListNode<T>{
        public LinkedListNode(T value)=>Value=value;
        public T Value{get;}
        public LinkedListNode<T> Next{get;internal set;}
        public LinkedListNode<T> Prev{get;internal set;}
    }
    //链表类
    public class LInkedLIst<T> : IEnumerable<T>
    {
        public LinkedListNode<T> First{get;private set;}//第一个对象
        public LinkedListNode<T> Last{get;private set;}//最后一个对象
        public LinkedListNode<T> AddLast(T node){
            var newNode=new LinkedListNode<T>(node);
            if(First==null){
                First=newNode;
                Last=First;
            }else{
                LinkedListNode<T> previous=Last;//previous指向倒数第二个对象
                Last.Next=newNode;//倒数第二个对象的Next指向最后一个对象
                Last=newNode;//last指向最后一个对象
                Last.Prev=previous;//最后一个对象Prev方法指向倒数第二个对象
            }
            return newNode;
        }

        //重写获取迭代方法
        public IEnumerator<T> GetEnumerator()
        {
            LinkedListNode<T> current=First;
            while(current!=null){
                yield return current.Value;//yield 语句创建一个枚举器的状态机
                current=current.Next;
            }
        }

        IEnumerator IEnumerable.GetEnumerator()=>GetEnumerator();
    }
    
//Main方法
       var list3=new LInkedLIst<int>();
       list3.AddLast(1);
       list3.AddLast(3);
       list3.AddLast(4);
       foreach(int i in list3){
         Console.WriteLine(i);
       }

                  public T GetE(){
	T doc=default;//T类型是值类型或引用类型
    return doc;

                  # 约束

                  概念：泛型类可以给T类型添加约束

                  约束 说明
                  where T:struct 对于结构约束，类型T必须是值类型
                  where T: class 类约束制定类型T必须是引用类型
                  where T: IFoo 指定类型T必须实现接口IFoo
                  where T:Foo 指定类型T必须派生自基类Foo
                  where T:new() 构造函数约束，制定类型T必须有一个默认构造函数
                  where T1:T2 这个约束可以指定，类型T1派生自泛型类型T2
                  public class MyClass<T>
where T:IFoo,new(){//T 类型继承TFoo接口 
    
}

                  # 继承

                  概念：泛型类型可以实现泛型接口，也可以派生自一个类，泛型类可以派生自泛型基类

                  要求必须重复接口的泛型类型，或者必须制定基类的类型。

                  派生类可以是泛型或非泛型类。可以定义一个抽象的泛型基类，他在派生类中用一个具体的类实现。

                  # 静态成员

                  概念： 泛型类的静态成员需特殊关注，泛型类的静态成员只能在类的一个实例中共享。

                  # 泛型接口

                  概念： 使用泛型可以定义接口，在接口中定义的方法可以带泛型参数。

                  # 协变和抗变

                  协变：如果泛型类型用out关键标注，泛型接口就是协变的。这也意味着返回值类型只能是T类型。

                  抗变：如果泛型类型用in关键字标注，泛型接口就是抗变的。这样，接口只能把泛型类型T用作其方法的输入

                  public class Shapes{

    public double Height{get;set;}

}

public interface IIndex<out T>
{
    T this[int index]{get;}
    int Count{get;}
}

    private static RectangleCollection _coll;

    _coll??(_coll=new RectangleCollection());

        get{

                return data[index];
        }
    }
    public int Count=>data.Length;

}

                  # 泛型方法

                  概念 除了定义泛型类型之外，还可以定义泛型方法，在泛型方法中，泛型类型用方法声明来定义，泛型方法可以在非泛型类中定义。

                  //泛型方法
public void Swap<T>(ref T x ,ref T y){
    T temp,
    temp=x;
    x=y;
    y=temp;
}
//调用泛型方法
int i=4;
int j=3;
Swap<int>(ref i,ref,j);//引用传递调换i和j的值

                  # 带委托的泛型

                  //委托泛型方法
public static T2 Accumulate<T1,T2>(IEnumerable<T1> source,Func<T1,T2,T2> action){
    T2 sum=default(T2);
    foreach(T1 item in source){
        sum=action(item,sum);
    }
    return sum;
}
//调用委托泛型方法
decimal amount =Algorithm.Accumulate<Account,decimal>(accounts,(item,sum)=>+=item.Balance);

                  # 总结

                  通过泛型类可以创建独立于类型的类，泛型方法是独立与类型的方法。接口、结构和委托也可以用泛型的方法创建，泛型引用了一种新的编程方式，它们都是类型安全的，泛型委托可以去除集合中的算法。
