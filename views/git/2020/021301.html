        git基础操作

        聂 2020-02-14 git

        参考穆雪峰Git教程 git官网

        提示

        代码或命令在 <> 内的需要根据自己情况修改

        # 本地安装及本地提交

        # 操作步骤

        1. 打开git官网选择下载，除了安装目录其他可默认选择
        2. 安装完成后，在选择一个盘符中创建一个文件，进入该文件加夹。如：D:\learngit
        3. 鼠标右击选择 git bash here
        4. git version 查看git安装版本
        5. git config --global user.name "<Your Name>" 配置你的名称
        6. git config --global user.email "<email@example.com>" 配置你的邮箱
        7. git init 初始化git环境
        8. git status 查看工作空间状态
        9. 打开目录创建learn.txt，建议使用 Notepad++ 创建保存
        10. git status txt文件显示红色
        11. git add learn.txt 添加到 stage暂存区
        12. git status txt文件显示绿色
        13. git commit -m "<提交说明>"
        14. git status 查看工作空间状态
        15. rm -rf .git 删除git环境

        说明

        git init 会创建一个 .git 隐藏文件夹可通过 ls -a 查看，rm -rf .git 删除文件则Git不会管理。

        git status 查看工作空间状态。 时刻查看文件状态

        git add <添加文档名称> 添加到stage暂存区域， git commit -m "<提交说明>" 提交到Repository版本库 。

        # 工作区和Repository版本库

        # 工作原理

        说明

        工作区：实际修改跟新文档的地方

        版本库： .git 隐藏文件夹下存储

        Stage: 中文译驿站、阶段，也可以理解为暂存区 add 命令添加，可多次添加同一个文件修改

        Master:中文译主人，可以理解为主分区

        # 操作步骤

        1. 选择进入文件夹 如 D:\learngit

        2. git init

        3. 创建文件 learn.txt,打开添加 < learn git > ，建议使用Notepad++打开

        4. git add learn.txt

        5. 向文件继续添加 < Text >

        6. git commit -m "<learn git>"

        7. git status

        8. git add learn.txt

        9. git commit -m "<Text>"

        10. git status

        11. git log 显示详细或git log --pretty=oneline 一行显示

        说明

        git log 详细提交信息

        git log --pretty=oneline 提交信息一行显示，黄色很长的代码为 commitId

        可用于恢复之前的版本，HEAD表示当前所在版本，master表示所在主分区

        # Git恢复管理

        # 操作步骤

        1. git reset --hard HEAD~1 返回上一个版本
        2. git log --pretty=oneline
        3. git reflog 查看命令记录
        4. git reset --hard < 7520 > 通过commit_id 到制定版本，可不用输入全
        5. 打开<learn.txt>，添加< moving > 保存
        6. git checkout -- <learn.txt> 恢复learn.txt更改部分
        7. 再次打开<learn.txt>,添加< moving > 保存
        8. git add learn.txt 添加stage
        9. git status
        10. git reset HEAD <learn.txt> 踢出 stage暂存区
        11. git status
        12. git checkout -- <learn.txt> 恢复learn.txt更改部分
        13. 删除本地<learn.txt> 文件
        14. git add learn.txt 或 git rm learn.txt
        15. git commit -m "<moving >" 提交删除

        命令解析

        git checkout -- <文件名>恢复仅工作区修改的文件

        git reset HEAD <文件名> 在执行 git checkout --<文件名>恢复恢复已经添加到stage中的， 或 git checkout HEAD <文件名> 直接恢复

        git reset --hard HEAD~1 （数字1代表上一个版本），已经提交到master中的文件恢复上一个版本，或 git reflog 命令查找 commit_id恢复制定版本

        git reflog 查看master分支的操作命令

        git add <文件名> 或 git rm < 文件名 > ，在 git commit -m "<提交说明>"提交到master，删除文件。

        注意

        没有被提交到master 中的文件git无法管理无法管理恢复
