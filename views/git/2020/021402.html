        git分支及远程库

        聂 2020-02-14 git

        # 分支操作

        1. 打开文件如：D:\learngit
        2. git init 初始化
        3. git checkout - b <dev> 创建进入 dev分支
        4. git branch 查看分支，当前分支会有一个 *表示
        5. 添加 < Readme.md > 文件
        6. git add Readme.md
        7. git commit -m "<dev branch>" 提交
        8. git checkout master 切换到主分支
        9. 文件 Readme.md 没有了
        10. git checkout master 切换到主分支
        11. git merge <dev> 合并分支
        12. git branch -d <dev> 删除分支

        说明

        git branch 查看分支命令， git branch <name> 创建分支,git checkout <name> 切换分支，git checkout -b <name> 创建+切换分支命令 ，git merge <name> 合并分支 git branch -d <name> 删除分支，删除没有合并的分支通过 git branch -D <name> 强行删除。

        注意

        git branch <name> 创建分支时根据当前分支创建。git branch -d <name> 删除分支不能在当前分支删除需要切换到其他分支。合并分支时出现冲突文件，通过 git status 查看冲突文件，处理冲突文件，在提交管理， git log --graph --pretty=oneline --abbrev-commit 查看分支情况。

        # 连接远程库

        1. ssh-keygen -t rsa -C "<youremail@example.com>" 创建SSH Key 然后一直回车
        2. 根据提示的目录一般在用户主目录下找到 .ssh 目录，里面有 id_rsa 和 id_rsa.pub两个文件。
        3. 复制 id_rsa_pub 文件内的公钥
        4. 登录GitHub，打开 “settings”，“SSH and GpG keys” 页面：
        5. 点击“Add SSH Key ”，填写Title,粘贴 id_rsa.pub 文件内容
        6. 在GitHub 创建一个仓库 <learngit>
        7. 在本地 learngit 库下运行命令 git remote add origin git@github.com:<你的用户名>/learngit.git
        8. git push -u origin master 将本地库所有内容推送到远程库

        说明

        git remote add origin git@github.com:<你的用户名>/<库名>.git 本地库远程提交远程库

        git clone git@github.com:<账户名>/<远程库名>.git 从远程库clone 到本地

        git push -u origin master 首次提交到远程库

        # 远程库提交

        git remote add origin git@github.com:<你的用户名>/<仓库名>.git 关联

        首次提交 git pull origin master --allow-unrelated-histories 下次 git pull origin master 解除 远程库关联 git remote remove origin

        克隆下载库 git clone git@github.com:<你的用户名>/<仓库名>.git 克隆下来的只有master分支的内容，如果添加dev分支 git checkout -b dev origin/dev

        注意

        提交到远程库 origin 中内容，是版本库master分区中的HEAD的部分

        # bug 修复

        stash说明

        临时保存分支 ,工作分支写到一半需要切换到其他分支修改bug，临时保存 新文件必须 add 才能储存

        git add 新文件 git stash 创建临时储存 切换其他分支，修复bug

        返回工作分支，恢复

        git stash list //查找临时分支 git stash apply //恢复不删除 git stash drop //删除分支 git stash pop //恢复并删除分支

        # 解决冲突问题

        1. git pull origin master 远程同步文件
        2. 解决本地文件冲突
        3. git push origin master 提交文件

        pull 操作相当于同步代码

        git fetch origin master git log -p master..origin/master git merge origin/master

        # 忽略文件

        • 忽略某些文件时，需要编写.gitignore 文件；
        • 在.gitignore写入忽略的文件 如 *.class文件、dist 文件夹
        • .gitignore文件本身要放到版本库，可以对.gitignore做版本管理！
